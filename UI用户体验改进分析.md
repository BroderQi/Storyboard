# UI/UX 用户体验改进分析

## 问题诊断

### 核心问题
**我们有两个输入端，但用户可能不理解它们的区别和用途：**

1. **视频上传 → 抽帧 → AI分析**（基于现有视频）
2. **文本生成分镜**（从零开始创作）

### 当前UI存在的问题

#### 问题1：两个入口不够明显，缺少引导
**现状：**
- **视频导入入口**：在左侧边栏 `ProjectSidebarView` 中，标题是"视频导入"
- **文本生成入口**：隐藏在某个按钮后面，需要点击 `ShowTextToShotDialog` 才能打开对话框
- **没有明确的"选择工作流"引导**

**问题：**
- 用户打开项目后，不知道应该选择哪种方式
- 两种方式的区别和适用场景不清晰
- 文本生成入口不够显眼，用户可能找不到

#### 问题2：命名和描述不够清晰
**现状：**
- 左侧边栏：**"视频导入"** → **"抽帧配置"** → **"AI 场景分析"**
- 对话框：**"文本生成分镜"**

**问题：**
- "视频导入"听起来只是导入视频，用户不知道这是一个完整的工作流
- "文本生成分镜"听起来像一个独立功能，用户不知道这和"意图分析"是类似的
- 缺少对两种方式的对比说明

#### 问题3：缺少"意图分析"步骤
**对比别人的工具：**
- 别人的工具有明确的"意图分析"步骤：
  - Core Message（为什么拍）
  - Target Audience（给谁看）
  - Key Plot Points（重点是什么）
  - Suggested Tone/Mood（整体基调）

**我们的现状：**
- **视频导入流程**：直接分析视频帧，没有理解用户的创作意图
- **文本生成流程**：直接生成分镜，也没有意图分析步骤

**问题：**
- AI不理解用户的创作目标，生成的分镜可能不符合预期
- 用户无法表达"我想要什么样的视频"

#### 问题4：工作流不够清晰
**现状：**
- 用户需要自己摸索：导入视频 → 抽帧 → AI分析 → 编辑 → 生成
- 没有明确的步骤指示
- 没有进度提示

**问题：**
- 新用户不知道下一步该做什么
- 容易迷失在复杂的功能中

## 对比分析：我们 vs 别人的工具

| 维度 | 别人的工具 | 我们的工具 | 差距 |
|------|-----------|-----------|------|
| **入口设计** | 明确的"开始新项目"，两个选项清晰可见 | 视频导入在左侧，文本生成入口不明显 | ⚠️ 需要改进 |
| **意图分析** | 有明确的意图分析步骤 | 没有意图分析 | ❌ 缺失 |
| **工作流引导** | 清晰的步骤：意图分析 → 脚本生成 → 剪辑建议 | 用户需要自己摸索 | ⚠️ 需要改进 |
| **功能命名** | "全维度导演脚本"、"AI剪辑建议" | "视频导入"、"AI场景分析" | ⚠️ 不够吸引人 |
| **用户引导** | 有详细的说明和示例 | 缺少引导 | ⚠️ 需要改进 |
| **视觉层次** | 清晰的卡片式布局，步骤明确 | 功能堆叠在左侧边栏 | ⚠️ 需要改进 |

## 改进方案

### 方案1：增加"工作流选择"引导页（推荐）

#### 设计思路
在项目创建后，首次进入项目时，显示一个**工作流选择页**，让用户明确选择：

```
┌─────────────────────────────────────────────────────────┐
│                  选择您的创作方式                          │
├─────────────────────────────────────────────────────────┤
│                                                           │
│  ┌──────────────────────┐  ┌──────────────────────┐    │
│  │  📹 基于现有视频      │  │  ✍️ 从零开始创作      │    │
│  │                      │  │                      │    │
│  │  适用场景：          │  │  适用场景：          │    │
│  │  • 已有视频素材      │  │  • 创意构思阶段      │    │
│  │  • 需要重新剪辑      │  │  • 脚本策划          │    │
│  │  • 分析现有内容      │  │  • 故事板设计        │    │
│  │                      │  │                      │    │
│  │  工作流：            │  │  工作流：            │    │
│  │  1. 导入视频         │  │  1. 描述创作意图     │    │
│  │  2. 智能抽帧         │  │  2. AI生成分镜       │    │
│  │  3. AI场景分析       │  │  3. 编辑调整         │    │
│  │  4. 编辑调整         │  │  4. 生成素材         │    │
│  │  5. 生成素材         │  │                      │    │
│  │                      │  │                      │    │
│  │  [开始导入视频]      │  │  [开始文本创作]      │    │
│  └──────────────────────┘  └──────────────────────┘    │
│                                                           │
│  💡 提示：两种方式可以混合使用，随时切换                 │
└─────────────────────────────────────────────────────────┘
```

#### 实现方式
1. 在 `MainViewModel` 中增加 `IsWorkflowSelectionVisible` 属性
2. 创建 `WorkflowSelectionView.axaml` 视图
3. 在 `MainWindow.axaml` 中，当项目打开且没有视频也没有分镜时，显示此页面

### 方案2：重新设计左侧边栏，增加"意图分析"卡片

#### 当前结构
```
左侧边栏：
├── 视频导入
├── 抽帧配置
├── AI 场景分析
└── 分镜统计
```

#### 改进后的结构
```
左侧边栏：
├── 📋 创作意图（新增）
│   ├── 创作目标
│   ├── 目标受众
│   ├── 视频基调
│   └── [保存意图]
│
├── 📹 方式一：基于视频
│   ├── 导入视频
│   ├── 抽帧配置
│   └── [执行抽帧]
│
├── ✍️ 方式二：文本创作
│   ├── [打开文本编辑器]
│   └── 说明：描述您的故事，AI自动生成分镜
│
├── 🤖 AI 分析
│   ├── [批量分析所有分镜]
│   └── 说明：智能识别场景、生成描述
│
└── 📊 分镜统计
    ├── 总数
    └── 总时长
```

#### 设计要点
1. **创作意图卡片**（新增）
   - 放在最顶部，强调"先想清楚要做什么"
   - 包含4个字段：创作目标、目标受众、视频基调、关键信息
   - 保存后，AI分析时会参考这些信息

2. **两种方式并列展示**
   - 清晰标注"方式一"和"方式二"
   - 每种方式都有简短说明
   - 视觉上平等，不偏向任何一种

3. **AI分析独立出来**
   - 不再隐藏在"视频导入"下面
   - 强调这是一个独立的步骤

## 视觉设计改进

### 当前问题
- 左侧边栏功能堆叠，视觉层次不清晰
- 两种创作方式没有明显区分
- 缺少视觉引导

### 改进建议

#### 1. 使用不同的颜色标识
- **方式一（基于视频）**：蓝色系 `#3b82f6`
- **方式二（文本创作）**：紫色系 `#8b5cf6`
- **AI分析**：绿色系 `#10b981`
- **创作意图**：橙色系 `#f59e0b`

#### 2. 使用图标增强识别
- 📹 视频导入
- ✍️ 文本创作
- 🤖 AI分析
- 📋 创作意图
- 📊 统计信息

#### 3. 卡片式布局
- 每个功能模块使用独立的卡片
- 卡片之间有明显的间距
- 重要功能使用更大的卡片

## 用户测试建议

**测试步骤：**

**测试目标：** 用户能否顺利完成"视频导入 → 抽帧 → AI分析"流程

**测试步骤：**
1. 给用户一个视频文件
2. 让用户完成分镜生成
3. 观察用户的操作路径
4. 记录用户的困惑点

**成功标准：**
- 80%的用户能在5分钟内完成整个流程
- 用户不需要询问"下一步该做什么"

**测试目标：** 用户能否理解"意图分析"的价值

## 总结

### 核心问题
1. **两个入口不够明显**：用户不知道有两种创作方式
2. **缺少意图分析**：AI不理解用户的创作目标
3. **工作流不清晰**：用户不知道下一步该做什么
4. **命名不够友好**：功能名称不够直观

### 解决方案
1. **重新设计左侧边栏**：两种方式并列展示，增加"创作意图"卡片
2. **改进文本生成对话框**：增加意图分析步骤
3. **增加工作流选择页**：首次进入时引导用户选择
4. **改进命名和文案**：使用更友好的语言

### 预期效果
- 新用户学习成本降低50%
- 用户满意度提升30%
- AI生成质量提升（因为有了意图分析）
- 两种创作方式的使用率更均衡
